# How To Pronounce

A TanStack Start React app for learning English pronunciation from real YouTube
examples. Enter a word, the app calls a transcript API, then plays matching
YouTube clips at the relevant timestamp.

## Tech Stack

- [Bun](https://bun.sh/) for dependency management and scripts
- [TanStack Start](https://tanstack.com/start) and TanStack Router
- React 19 with TypeScript
- TanStack Query for API requests
- Tailwind CSS 4 for styling
- Vite, Nitro, and Cloudflare Workers deployment via Wrangler

## Requirements

- Bun
- A transcript API endpoint that supports `GET /api?q=<word>`
- Cloudflare Wrangler access for deployment

## Environment

Create a local environment file from the example:

```bash
cp .env.example .env
```

Set the API base URL:

```bash
VITE_API_URL=https://your-api.example.com
```

The frontend calls `${VITE_API_URL}/api?q=<query>` and expects:

```ts
{
  message: string;
  query: string;
  transcripts: {
    text: string;
    videoId: string;
    start: number;
    duration?: number;
  }[];
}
```

## Development

Install dependencies:

```bash
bun install
```

Start the local dev server on port `3000`:

```bash
bun run dev
```

Run checks:

```bash
bun run test
bun run lint
bun run format
```

Auto-format and apply ESLint fixes:

```bash
bun run check
```

## Build and Deployment

Create a production build:

```bash
bun run build
```

Build for Cloudflare Workers:

```bash
bun run build:cf
```

Deploy to Cloudflare:

```bash
bun run deploy
```

For local deployment builds that should ignore `VITE_API_URL`, use:

```bash
bun run deploy:local
```

Cloudflare configuration lives in `wrangler.jsonc`. The current custom domain is
`say.siwakasen.dev`.

## Project Structure

```text
src/
  components/                  Reusable UI components
  integrations/tanstack-query/  Query provider and devtools
  interfaces/                  Shared TypeScript interfaces
  routes/                      TanStack file-based routes
  routeTree.gen.ts             Generated router tree
  router.tsx                   Router setup
  styles.css                   Global styles
public/                        Static assets and PWA metadata
```

Do not edit `src/routeTree.gen.ts` manually; it is generated by TanStack Router.

## Testing

This project uses Vitest with React Testing Library available. Tests can be
placed next to the code they cover using names such as `Component.test.tsx` or
`feature.test.ts`.

```bash
bun run test
```

## Formatting and Linting

Formatting is managed by Prettier and linting by the TanStack ESLint config.
The project uses 2-space indentation, semicolons, single quotes, trailing
commas, and an 80-character print width.
